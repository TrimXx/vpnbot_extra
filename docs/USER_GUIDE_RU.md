# VPnBot: руководство пользователя (RU)

Это практическая инструкция по текущему функционалу бота: что за что отвечает, как выдавать подписки, как работает HWID/runtime mode, как управлять устройствами и трафиком.

## 1. Что входит в систему

- `app/bot.php` — Telegram-бот (админ-панель и управление пользователями).
- `app/subscription.php` — веб-страница подписки клиента.
- `config/xray.json` — реальные клиенты Xray и параметры устройств.
- `config/pac.json` — глобальные настройки бота/подписок/HWID.

## 2. Основные сущности

- **Профиль (подписка)** — родительский клиент Xray.
- **`subscription_id`** — стабильный идентификатор подписки в ссылках.
- **`id` (UUID)** — UUID клиента Xray.
- **HWID** — идентификатор устройства от клиента.
- **Runtime-устройство** — дочерний Xray-клиент (`device_parent_id`, `device_hwid`, `device_runtime`).
- **Пароль удаления устройства** — хранится как `device_delete_password_md5` у родительского клиента.

## 3. Как работает подписка

### Обычный режим (runtime OFF)

- Используется родительский UUID.
- HWID проверяется при получении/обновлении подписки.
- На уровне подключений Xray нет полной модели "1 устройство = 1 UUID".

### Runtime mode (runtime ON)

- Для каждого HWID создается отдельный дочерний UUID.
- Фактически получается модель "1 устройство = 1 профиль/UUID".
- Лимит устройств = число активных HWID (дочерних клиентов).
- Используется ленивая миграция: старые ссылки продолжают работать.

## 4. Глобальные и локальные флаги

В `pac.json`:

- `hwid_limit_enabled` — включен ли контроль HWID.
- `hwid_runtime_mode_enabled` — глобальный runtime mode.
- `hwid_device_count` — дефолтный лимит устройств.

В карточке конкретного пользователя:

- `hwid_disabled` — отключить HWID для конкретной подписки.
- `hwid_limit` — индивидуальный лимит устройств.
- `hwid_runtime_mode` — override runtime для подписки.

Приоритет: если у подписки задан собственный runtime-флаг, применяется он.

## 5. Что есть в карточке пользователя (Telegram)

Раздел `Xray -> пользователь`:

- Ссылки и импорт для клиентов (`v2ray`, `sing-box`, `mihomo`).
- Переключение ON/OFF.
- Срок действия (timer).
- Сброс статистики.
- Выбор шаблонов.
- QR-коды.
- Переименование/удаление пользователя.
- Переход в HWID-меню.
- Кнопка сброса пароля удаления устройств (`reset delete password`).

## 6. HWID-меню пользователя

В разделе устройств можно:

- Смотреть список обнаруженных устройств (HWID + метаинфо).
- Удалять конкретные устройства.
- Менять лимит устройств.
- Переключать runtime mode для конкретной подписки.
- Возвращать логику на "по умолчанию" (global).

## 7. Веб-страница подписки (`/sub?id=...`)

Клиенту доступны:

- Ссылки/QR для подключения.
- Список подключенных устройств (компактный блок).
- Трафик по устройству в формате:
  - `⬇ Download`
  - `⬆ Upload`
- Удаление устройства кнопкой `🗑️`.

## 8. Пароль удаления устройства

На странице подписки:

- Кнопка `🔐` в хедере:
  - если пароль не установлен — "Задать пароль";
  - если установлен — "Сменить пароль".
- При смене пароля требуется текущий пароль.
- Для удаления устройства обязательно вводить пароль.
- В интерфейсе есть примечание: если пароль забыт, обращаться в поддержку.

В Telegram:

- Админ может сбросить пароль удаления устройства для подписки (кнопка в профиле пользователя).

## 9. Трафик

- Статистика привязана к стабильному ключу клиента (`id`), а не только к индексу массива.
- Благодаря этому корректно считается трафик по устройствам в runtime-модели.

## 10. Типовой сценарий выдачи

1. Создать пользователя в Xray-разделе бота.
2. Установить срок/лимит/шаблон.
3. Включить runtime (глобально или локально), если нужен строгий контроль устройств.
4. Выдать ссылку подписки клиенту.
5. Клиент добавляет подписку в приложение.
6. Проверить появление устройства в HWID-списке.
7. При необходимости удалить устройство или сбросить пароль удаления.

## 11. Частые проблемы и диагностика

- **"Subscription not found or disabled"** — неверный ID, подписка выключена или истек срок.
- **`outbound index missing`** — проблема шаблона (несоответствие `proxies`/`outbounds`).
- **Дубликаты пользователей в Xray** — проверить `xray.json` на повторяющиеся device-клиенты.
- **Нет устройств в списке** — клиент не передает HWID или HWID-контроль отключен.

## 12. Безопасность

- Не хранить реальные пароли, ключи, токены в Git.
- Конфигурации с чувствительными данными держать в `.gitignore`.
- Если секрет уже попал в публичный репозиторий:
  - удалить его из истории;
  - обязательно ротировать (сменить) секреты.
